# 工时记录软件 - 需求规格说明书

## 1. 项目概述
开发一款轻量级 Web 端工时记录工具，支持多用户数据隔离（用户切换无密码）。用户可建立"项目-任务"二级结构，以分钟为单位记录工时，并通过日/周/月视图直观查看时间分布。核心目标为帮助用户精确回答"时间花在哪里"。

---

## 2. 用户切换模块
> **设计理念**：本地工具类应用，无需密码认证，通过用户选择快速切换身份。

| 需求编号 | 需求描述 |
| :--- | :--- |
| AUTH-01 | 首页展示用户列表，点击用户即可"登录"（进入系统），无需密码。 |
| AUTH-02 | 支持创建新用户（仅需输入用户名，用户名不可重复）。 |
| AUTH-03 | 每个用户登录后只能查看和操作自己的数据，用户之间数据**完全隔离**。 |
| AUTH-04 | 导航栏提供"切换用户"按钮，点击后退出当前用户并返回用户选择页。 |

---

## 3. 项目管理模块
> **约束**：所有删除操作均采用**软删除**机制（数据底层保留，界面不可见），工时记录仍关联原项目 ID 供历史统计。

| 需求编号 | 需求描述 |
| :--- | :--- |
| PROJ-01 | 用户可以**创建**项目，创建时**必须**填写名称（描述为**选填**，列表页若为空显示“--”）。 |
| PROJ-02 | 用户可以**编辑**项目名称和描述。 |
| PROJ-03 | 用户可以**删除**项目（软删除）。删除非默认项目时，其下所有任务的工时记录统一转移至全局“默认项目/默认任务”，随后执行软删除。 |
| PROJ-04 | **强制默认规则**：新用户创建成功后，系统**自动**创建一个"默认项目"（底层标记 `is_default=1`）。 |
| PROJ-05 | 允许用户重命名“默认项目”的显示名称，但底层标记保留。**“默认项目”严禁删除**（删除按钮置灰/隐藏）。 |

---

## 4. 任务管理模块
> **约束**：所有删除操作均采用**软删除**机制（数据底层保留，界面不可见）。

| 需求编号 | 需求描述 |
| :--- | :--- |
| TASK-01 | 用户可以在**指定项目下**创建任务，创建时**必须**填写名称（描述为**选填**，列表页若为空显示“--”）。 |
| TASK-02 | 用户可以**编辑**任务名称和描述。 |
| TASK-03 | 用户可以**删除**任务（软删除），删除时触发工时转移逻辑（详见第 8 节）。 |
| TASK-04 | **强制默认规则**：每个项目（含默认项目）被创建时，系统**自动**在该项目下创建一个“默认任务”（底层标记 `is_default=1`）。 |
| TASK-05 | 允许用户重命名“默认任务”的显示名称，但底层标记保留。**“默认任务”严禁删除**（删除按钮置灰/隐藏），确保每个项目始终有兜底任务。 |

---

## 5. 工时记录模块
> **工时单位**：最小单位为 **1 分钟**，输入正整数。

| 需求编号 | 需求描述 |
| :--- | :--- |
| TIME-01 | 记录工时时，用户**必须**同时指定：**日期 + 项目 + 任务 + 工时数值（分钟）**。 |
| TIME-02 | 允许在同一天、同一项目、同一任务下存在多条记录（支持分段记录）。 |
| TIME-03 | 用户可以对已录入的记录进行**编辑**。编辑时**允许**修改日期、换绑项目/任务、修改时长。 |
| TIME-04 | 用户可以**物理删除**某条工时记录（直接从数据库中移除）。 |
| TIME-05 | **合法校验**：编辑或新增记录时，日期**不可晚于今天**（禁止记录未来工时）。 |

## 6. 数据视图模块（日/周/月）
> **交互要求**：视图切换通过选项卡一键完成，切换点击不超过 2 次。

| 需求编号 | 需求描述 |
| :--- | :--- |
| VIEW-01 | **日视图**：展示选定日期的工时记录明细列表，并按项目/任务汇总当日总工时。 |
| VIEW-02 | **周视图**：展示选定周（周一至周日）每天的工时汇总柱状图/列表，支持**点击某天穿透**查看该天的明细记录。 |
| VIEW-03 | **月视图**：展示选定月份的每日工时汇总（日历热力图或列表形式），支持**点击某天穿透**查看该天的明细记录。 |
| VIEW-04 | 所有视图的工时汇总数据，必须以 **“X小时X分钟”**（如“1小时30分钟”）或纯分钟数清晰展示，确保可读性。 |

## 7. 数据导出模块（CSV）
| 需求编号 | 需求描述 |
| :--- | :--- |
| EXP-01 | 支持将当前用户的工时记录**导出为 CSV 格式**文件。 |
| EXP-02 | **导出范围**：导出时，必须提供日期选择器，由用户**手动指定导出范围的起始日期和结束日期**（不自动跟随当前视图范围）。 |
| EXP-03 | CSV 文件**必须**包含以下字段列：`日期, 项目名称, 任务名称, 工时(分钟), 记录ID`。 |
| EXP-04 | 导出功能入口在视图页面或独立设置页提供“导出”按钮触发。 |

## 8. 核心业务逻辑（工时转移机制）
> **本规则是系统的核心复杂逻辑，必须严格实现。**

### 8.1 任务删除时的工时转移
- **触发**：用户删除某任务（非默认任务）。
- **转移目标**：系统**自动**将该任务下的所有历史工时记录，批量转移至**同一项目下的“默认任务”**。
- **兜底方案**：若因异常情况该项目下无默认任务，则转移至全局“默认项目”下的“默认任务”。
- **用户交互**：删除前，界面**必须弹窗明确提示**用户：“该任务下共有 X 条工时记录，将自动转移至本项目的默认任务【任务名称】”，用户确认后方可执行删除。

### 8.2 项目删除时的工时转移
- **触发**：用户删除某项目（仅限非默认项目）。
- **转移目标**：系统将其下所有任务的工时记录，**统一转移**至全局“默认项目”下的“默认任务”。
- **后续操作**：转移完成后，对该项目及其下所有任务执行软删除（界面不可见）。

## 9. 非功能性约束与技术栈
| 类别 | 具体要求 |
| :--- | :--- |
| **前端交互** | Web 交互，界面简洁清晰，视图切换流畅。 |
| **数据库** | 必须使用 **SQLite** 作为嵌入式数据库存储。 |
| **性能指标** | 月视图加载 31 天数据时，响应时间 **< 2 秒**。 |
| **技术栈** | Flask + Jinja2 + sqlite3 + Chart.js，Python 后端，服务端渲染。 |

## 10. 附录（数据字典简要定义）
系统至少包含以下核心数据实体（供 Agent 参考设计）：
- **用户**：ID, 用户名(唯一), 创建时间
- **项目**：ID, 用户ID(外键), 名称, 描述, 是否默认, 软删除标记, 创建时间
- **任务**：ID, 项目ID(外键), 名称, 描述, 是否默认, 软删除标记, 创建时间
- **工时记录**：ID, 用户ID(外键), 项目ID(外键), 任务ID(外键), 日期, 工时(分钟), 创建/更新时间
